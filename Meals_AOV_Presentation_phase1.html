1. The metric and the AOV gap

Core AOV (meals) is computed from public_edw_business_mart_live.revenue_dashboard as:

SUM(order_item_revenue_excl_sales_tax + shipping_revenue_excl_sales_tax) / SUM(orders_count) WHERE entity_code = 'HF-CA' AND order_item_type = 'Mealboxes' GROUP BY hellofresh_week

This formula reconciles to the tracker AOV Core row to the cent for W1–W18 and within $0.41 for W19–W25 (CSV freshness lag).

2. Phase 1 framework

Core recipe prices and shipping fees are fixed in 2026 YTD. Per-box Core AOV can only move via one or more of these three mechanisms:

  1. Seamless downgrade share changes — the % of customers whose shipped box was auto-shrunk via the seamless flow.
  2. Seamless upgrade share changes — the % of customers whose shipped box was auto-grown via the seamless flow.
  3. Original plan mix changes — the distribution of plans (people × recipes) customers originally select.

This document reports weekly trends and cohort breakdowns for each mechanism, plus a Laspeyres calculation quantifying how much of the AOV gap is mechanically attributable to plan-mix shift.

Convention. Plan codes use XxY = X recipes × Y people in both the seamless table (original_box_type) and the KPI tracker (PBxy). Verified empirically against the SKU pattern CA-{base}-{recipes}-{people}-0 in facts_recipes_ordered_enriched.

Direction reading for charts: a decline in downgrade % is favorable for AOV (good); a decline in upgrade % is unfavorable for AOV (bad).
Phase 2 scope (out of this document): AOR analysis for seamless-downgraded customers, pause rate, cancellation rate, interpretation, and recommendations.

7. Plan-mix contribution to AOV — summary

Question: Of the observed AOV decline, how much is mechanically caused by customers selecting smaller original plans on average — independent of any seamless behavior?

Method (one line): hold each plan's $/box at its W1–W5 average, then re-compute AOV using each week's actual mix of original plans. The only thing changing is which plans customers selected.

MetricValue
Baseline (W1–W5) mix-only AOV$105.44
Deep gap (W19–W24) mix-only AOV$104.87
Mix-only Δ from baseline to deep gap−$0.57/box
Observed tracker AOV gap (W19–W24 avg vs budget)−$0.74/box
Mix contribution as % of observed gap~77%

Plan-mix shift accounts for ~$0.57/box of the ~$0.74/box observed gap. The remaining ~$0.17/box is attributable to changes in seamless behavior (mainly weakening upgrades, per §4) and small calculation differences vs the canonical source.

Full formula, per-plan baseline $/box, weekly mix-only AOV chart and table are in the Appendix — Laspeyres calculation detail.

Appendix — Laspeyres calculation detail

How to read the chart below in plain language

For each week, we ask a simple "what-if": if every plan still earned the same $/box it earned in W1–W5 (e.g., a 2p×3r box still earns ~$78, a 4p×5r still earns ~$200, a 6p×5r still earns ~$300), what would AOV be — given only the actual change in which original plans customers selected that week?

Concretely, the calculation:

  • Holds constant: each plan's baseline $/box. This $/box already reflects baseline seamless behavior (downgrades and upgrades that were happening in W1–W5). It is not a "remove seamless" scenario.
  • Lets vary: the share of customers on each original plan, week by week.

Reading the result: if mix-only AOV drops by $0.57 from baseline to deep gap, that means $0.57 of the AOV decline is purely from customers selecting smaller original plans on average — even if seamless behavior had stayed exactly the same as in baseline.

Note — this is not "AOV without seamless". That would be a different calculation, using each customer's pre-seamless original-plan price instead of the realized price. That counterfactual is a Phase 2 question.

Formula

ΔAOV_mix = Σ_p (share_p^t − share_p^baseline) × AOV_p^baseline where: share_p^t = share of plan p in week t's original-plan mix share_p^baseline = share of plan p averaged over baseline (W1–W5) AOV_p^baseline = per-plan revenue per box, baseline avg

Computed from facts_recipes_ordered_enriched over W1–W5. Each value is the average revenue per box for that plan during the baseline window. These are the prices that stay locked when we vary the mix.
